# COMP-005: Adjacent High-Performance UIs

**Date:** 2026-04-12
**Status:** DRAFT
**Author:** Rob Jones (Design Director)

---

## Why Adjacent?
Recruitment software is historically slow, bloated, and "legacy-feeling." To build the next generation, we must look at high-performance tools in **Issue Tracking, Communication, and DevTools.**

---

## 1. Linear (The Standard for Speed)
**Core Lesson:** Speed is a feature. Keyboard-first navigation and clear, semantic hierarchy.
**Sinta Application:**
- Command menu (Cmd+K) for quick actions.
- A "Density" setting (Compact vs. Relaxed).
- High-contrast sidebar that separates navigation from workspace.

## 2. Superhuman (The Standard for Focus)
**Core Lesson:** Designing for the "Flow State." Minimal distraction, single-task focus.
**Sinta Application:**
- "Live Interview" mode that hides all global navigation and sidebars.
- Large, clear visual indicators that don't require searching for information.

## 3. GitHub PR Review (The Standard for Density)
**Core Lesson:** Co-located context. The diff, the comments, and the file tree in one view.
**Sinta Application:**
- The **Three-Column Review** flow.
- "Hover-to-reveal" metadata that keeps the primary view clean.
- Sticky headers for context (e.g., candidate name and role always visible).

## 4. Loom (The Standard for Video Interaction)
**Core Lesson:** Timestamp-as-a-unit-of-value. Interacting with video content through the timeline.
**Sinta Application:**
- Interactive transcripts where clicking a word jumps the video to that moment.
- Visual "heatmaps" on the video timeline showing where reactions/tags occurred.

## 5. Notion (The Standard for Block Architecture)
**Core Lesson:** Modular content. Everything is an object that can be moved or configured.
**Sinta Application:**
- The **Builder** stage cards as "blocks" with their own internal configuration.
- Flexible "Scorecard" layouts that can be customized per role.
